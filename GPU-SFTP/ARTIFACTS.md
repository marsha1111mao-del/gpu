# GPU-SFTP Artifact Policy

`GPU-SFTP` is a lightweight source/config/deploy-layout repository. It keeps
the files that describe the VM runtime layout, tests, and launch scripts, but
it does not version large or generated runtime payloads.

The runtime payloads are intentionally ignored by Git and are rebuilt or
restored by the surrounding `/home/mzh/gpu` workspace.

## Tracked

- `firecracker-bins/configs/`: Firecracker JSON and broker TOML configs.
- `firecracker-bins/scripts/`: VM launcher and rootfs helper scripts.
- `tests/`: userspace smoke-test source and build scripts.
- `artifacts/dtb/trans_dtb.sh`: DTB-to-DTS helper.
- `panthor_fw/mali_csffw.bin`: Mali CSF firmware input blob used when building
  Panfrost rootfs images. This is not generated by this workspace.
- `linux-host-kernel/cp_host_kernel.sh`: helper script for host kernel payloads.
- `.gitkeep` files that preserve expected runtime/log directories.

## Ignored Runtime Artifacts

### Firecracker and vmshm binaries

Ignored paths:

```text
firecracker-bins/bin/firecracker
firecracker-bins/bin/vmshm-broker
firecracker-bins/bin/vmshm-client-test
```

Rebuild/install from the main workspace:

```bash
cd /home/mzh/gpu
./scripts/build/build-firecracker-runtime.sh
```

The script builds `firecracker/Firecracker-CCA-MZH` and
`firecracker/vmshm-broker`, then installs the binaries into
`GPU-SFTP/firecracker-bins/bin/`.

### Panthor IOCTL smoke binary

Ignored paths:

```text
tests/panthor-ioctl-smoke/panthor_ioctl_smoke
firecracker-bins/bin/panthor_ioctl_smoke
```

Rebuild/install:

```bash
cd /home/mzh/gpu
./scripts/build/build-panthor-ioctl-smoke.sh
```

The workspace build script uses the Panthor UAPI from
`/home/mzh/gpu/Linux-Guest-GPU/include/uapi/drm` when available and installs the
static aarch64 test binary into `GPU-SFTP/firecracker-bins/bin/`. The
`GPU-SFTP/tests/panthor-ioctl-smoke/build.sh` wrapper remains for remote or
legacy callers.

### vmshm demo binary

Ignored paths:

```text
tests/vmshm-test/vmshm_demo
firecracker-bins/bin/vmshm_demo
```

Rebuild/install:

```bash
cd /home/mzh/gpu
./scripts/build/build-vmshm-demo.sh
```

The workspace build script installs the static aarch64 demo binary into
`GPU-SFTP/firecracker-bins/bin/`, which is the path consumed by
`firecracker-bins/scripts/tools/copy-vmshm-test.sh`. The
`GPU-SFTP/tests/vmshm-test/compile.sh` wrapper remains for remote or legacy
callers.

### GLES compute smoke binary

Ignored path:

```text
firecracker-bins/bin/gles-compute-smoke
```

This payload is built on the remote GPU host by the passthrough performance
workflow:

```bash
cd /home/mzh/gpu
./scripts/run/run-host-vs-passthrough-gles-perf.sh --host-rootfs-userspace
```

The script compiles `GPU-SFTP/tests/gpu-compute-smoke/gles_compute_smoke.c` on
the remote host, installs the binary into remote
`/root/GPU-SFTP/firecracker-bins/bin/`, and injects it into the selected rootfs.

### Shared and passthrough guest kernels

Ignored paths:

```text
firecracker-bins/kernels/shared/client/Image
firecracker-bins/kernels/shared/proxy/Image
firecracker-bins/kernels/passthrough/Image
```

Rebuild shared proxy/client kernels:

```bash
cd /home/mzh/gpu
./scripts/build/build-guest-vmshm-kernels.sh
```

Rebuild the passthrough kernel:

```bash
cd /home/mzh/gpu
./scripts/build/build-guest-passthrough-kernel.sh
```

Both scripts keep incremental kernel build outputs under
`Linux-Guest-GPU/out/` and install the resulting `Image` files into the
`GPU-SFTP/firecracker-bins/kernels/` layout.

### Host kernel deployment payloads

Ignored paths:

```text
linux-host-kernel/Image
linux-host-kernel/image_map_*
linux-host-kernel/lib/
linux-host-kernel/modules-staging/**/*.ko
```

Produce/deploy from the main workspace:

```bash
cd /home/mzh/gpu
./scripts/deploy/deploy-host-kernel-and-test.sh --skip-firecracker-build --skip-tests
```

For selected module deployment, use the script's `--host-modules` option. Full
module trees are local deployment payloads and should not be committed.

### Rootfs images

Ignored paths:

```text
firecracker-bins/rootfs/rootfs.ext2
firecracker-bins/rootfs/rootfs-panfrost.ext4
firecracker-bins/rootfs/mounts/
firecracker-bins/rootfs/work/
```

`rootfs-panfrost.ext4` can be rebuilt on the remote host from the tracked GLES
smoke/rootfs scripts:

```bash
ssh root@192.168.137.10 \
  'cd /root/GPU-SFTP/tests/gpu-compute-smoke && ./build-panfrost-rootfs-from-host-remote.sh'
```

or, for a debootstrap-based rebuild:

```bash
ssh root@192.168.137.10 \
  'cd /root/GPU-SFTP/tests/gpu-compute-smoke && ./build-panfrost-rootfs-remote.sh'
```

The lightweight `rootfs.ext2` used by vmshm tests is a runtime image expected
under `firecracker-bins/rootfs/`. It is excluded from ordinary sync and should
be restored from the local/remote artifact cache or regenerated by the
Firecracker/rootfs setup workflow before running tests that require it.

### Firecracker DTB/DTS dumps

Ignored paths:

```text
artifacts/dtb/*.dtb
artifacts/dtb/*.dts
```

Firecracker can write these via `machine-config.dump_fdt_path`, and
`artifacts/dtb/trans_dtb.sh` converts DTB dumps to readable DTS when needed.

### Logs

Ignored paths:

```text
log/**
```

The repository keeps only log directory skeletons. Test logs are local run
artifacts and should remain outside Git history.

## Sync Notes

The top-level test/deploy scripts under `/home/mzh/gpu/scripts/` sync this tree
to `root@192.168.137.10:/root/GPU-SFTP` and intentionally exclude logs and
rootfs images unless a rootfs sync is explicitly requested.

Use those scripts rather than ad hoc `rsync` so the remote semantic layout
migration in `/home/mzh/gpu/scripts/lib/gpu_sftp_layout.sh` is applied.
