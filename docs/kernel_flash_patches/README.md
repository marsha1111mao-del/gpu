# OpenCCA Raw Kernel Flash Patch Bundle

Status note for this workspace, 2026-06-11: these patches are retained as
reference material only. The working implementation in `opencca/` uses SD-card
storage ID `2` and a 128 MiB ext4/extlinux `kernel` boot partition at p3
(`0x8000`), with rootfs at p4 (`0x48000`). Do not apply these raw FIT patches
blindly to the current workspace.

This directory contains mail-style Git patches for importing the raw kernel
and rootfs flashing flow into another OpenCCA workspace.

The patches are split by component repository. Apply each patch from the root
of the matching component repo, not from the top-level OpenCCA workspace.

## Contents

| Component repo | Patch | Purpose |
| --- | --- | --- |
| `opencca-flash/` | `opencca-flash/0001-opencca-support-raw-kernel-and-rootfs-flashing.patch` | Adds `mmc-kernel`, `mmc-rootfs`, GPT layout validation, rootfs image selection, tests, and the older prebuilt image converter. |
| `opencca-build/` | `opencca-build/0001-opencca-build-raw-kernel-image.patch` | Makes `linux.mk kernel` build `snapshot/kernel.img`, a padded FIT image containing the Linux `Image` and Rock 5B DTB. |
| `u-boot/` | `u-boot/0001-opencca-boot-from-raw-kernel-slot.patch` | Updates the Rock 5B U-Boot fragment to read and boot the raw kernel slot at sector `0x8000`. |
| `debian-image-recipes/` | `debian-image-recipes/0001-opencca-add-raw-kernel-partition-to-image.patch` | Adds the raw `kernel` partition to the generated image and moves `root` to sector `294912` (`0x48000`). |

## Import

Copy this whole `kernel_flash_patches/` directory to the root of the target
OpenCCA workspace. Then run:

```sh
cd opencca-flash
git am -3 ../kernel_flash_patches/opencca-flash/*.patch

cd ../opencca-build
git am -3 ../kernel_flash_patches/opencca-build/*.patch

cd ../u-boot
git am -3 ../kernel_flash_patches/u-boot/*.patch

cd ../debian-image-recipes
git am -3 ../kernel_flash_patches/debian-image-recipes/*.patch
```

If a patch conflicts, inspect the conflict in that component repo, resolve it,
then continue with:

```sh
git am --continue
```

To abandon a failed import in the current component repo:

```sh
git am --abort
```

## Resulting Image Layout

The compatible eMMC image layout is:

| Partition | Start sector | Notes |
| --- | ---: | --- |
| `loader1` | `0x40` | Rockchip loader area |
| `loader2` | `0x4000` | U-Boot FIT area |
| `kernel` | `0x8000` | Raw 128 MiB kernel slot, `0x40000` sectors |
| `root` | `0x48000` | Rootfs partition, sector `294912` |

The raw kernel slot contains `snapshot/kernel.img`, generated by
`opencca-build/buildconf/linux.mk`. U-Boot loads that slot with:

```sh
mmc read 0x50000000 0x8000 0x40000
bootm 0x50000000
```

## Build And Flash Flow

After importing the patches, rebuild Linux and firmware:

```sh
cd opencca-build/buildconf
./linux.mk kernel
make -f firmware_opencca.mk build
```

`./linux.mk kernel` must produce:

- `snapshot/Image`
- `snapshot/rk3588-rock-5b.dtb`
- `snapshot/kernel.itb`
- `snapshot/kernel.img`

`make -f firmware_opencca.mk build` must rebuild U-Boot so the updated
`rk3588_fragment.config` is packed into `snapshot/u-boot.itb`.

Initialize boards that still use the older image layout by flashing either a
newly generated full image from `debian-image-recipes/` or a converted old
prebuilt image:

```sh
python3 opencca-flash/convert_prebuilt_image.py \
  ./opencca-image-rockchip-rock5b-rk3588.img \
  ./opencca-image-rockchip-rock5b-rk3588-compatible.img \
  --kernel ./snapshot/kernel.img
```

After the board has the compatible GPT layout, incremental updates can be:

```sh
ssh pi '~/opencca-flash/flash.sh mmc-kernel'
ssh pi '~/opencca-flash/flash.sh mmc-rootfs'
```

`mmc-kernel` flashes only `idbloader.img`, `u-boot.itb`, and `kernel.img`.
`mmc-rootfs` flashes only the rootfs image. It uses
`snapshot/opencca-rootfs-wip.ext4` by default, or `OPENCCA_ROOTFS_IMAGE` when
set.

## Validation

Run the local flash-side checks after import:

```sh
cd opencca-flash
python3 test_convert_prebuilt_image.py
python3 test_flash_script.py
python3 test_validate_gpt_kernel_slot.py
python3 -m py_compile convert_prebuilt_image.py validate_gpt_kernel_slot.py
```

For hardware validation, use the existing three-machine flow: build on the x86
OpenCCA host/container, flash through the Raspberry Pi flash server (`ssh pi`),
and boot/test on the Rock 5B board (`ssh opencca` after Linux boots).

The current automated flow is:

```sh
python3 run_test.py --flash-command mmc-kernel
```

If the board needs a one-time layout initialization first:

```sh
python3 run_test.py \
  --init-image ./opencca-image-rockchip-rock5b-rk3588-compatible.img \
  --flash-command mmc-kernel
```

Check `test_logs/latest/test.log`, `test_logs/latest/serial.log`, and the
pulled `test_logs/latest/minicom.txt` after a run.

## Important Compatibility Note

The raw kernel and rootfs modules must match. Do not change the kernel image
version string to hide a module mismatch. Rebuild or update the rootfs so
`/lib/modules/$(uname -r)` exists for the booted kernel.
