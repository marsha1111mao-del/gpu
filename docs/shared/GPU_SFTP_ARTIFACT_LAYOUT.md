# GPU-SFTP Artifact Layout

This document is the canonical layout note for `/home/mzh/gpu/GPU-SFTP` and
the mirrored remote tree at `/root/GPU-SFTP`.

## Top-Level Tree

```text
GPU-SFTP/
  artifacts/
    dtb/
  firecracker-bins/
    bin/
    configs/
    kernels/
    rootfs/
    scripts/
  linux-host-kernel/
  log/
  panthor_fw/
  tests/
```

`GPU-SFTP/firecracker-bins/` is a reusable artifact store. Do not treat it as
disposable state during smoke retests. Kernel output directories under
`Linux-Guest-GPU/out/` are also incremental build state and should be preserved
unless the user explicitly asks for a clean rebuild or the build tree is
corrupted.

## Runtime Artifacts

```text
firecracker-bins/bin/
  firecracker
  vmshm-broker
  vmshm-client-test
  panthor_ioctl_smoke
  gles-compute-smoke
```

Firecracker and broker builds install into `bin/` through
`scripts/build/build-firecracker-runtime.sh`. The Panthor IOCTL smoke build installs
`panthor_ioctl_smoke` into the same directory.

## Configs

```text
firecracker-bins/configs/shared/vmshm-1client/
  broker-config.toml
  client-vm-config.json
  proxy-vm-config.json

firecracker-bins/configs/shared/vmshm-2client/
  broker-config.toml
  client0-vm-config.json
  client1-vm-config.json
  proxy-vm-config.json

firecracker-bins/configs/passthrough/
  gpu-panfrost-vm-config.json
  gpu-passthrough-vm-config.json
```

Shared configs are generated by the semantic installers under
`firecracker-bins/scripts/shared/...`. Passthrough configs live separately under
`configs/passthrough/`.

## Kernels

```text
firecracker-bins/kernels/shared/client/Image
firecracker-bins/kernels/shared/proxy/Image
firecracker-bins/kernels/passthrough/Image
firecracker-bins/kernels/passthrough/backups/
```

Use `scripts/build/build-guest-vmshm-kernels.sh` for the shared
client/proxy role kernels. Use
`scripts/build/build-guest-passthrough-kernel.sh` for the single passthrough
guest kernel.

## Rootfs Images

```text
firecracker-bins/rootfs/rootfs.ext2
firecracker-bins/rootfs/rootfs-panfrost.ext4
firecracker-bins/rootfs/mounts/
firecracker-bins/rootfs/work/
```

Rootfs images are excluded from ordinary sync by default. Sync base images only
when a test requires a new base image, when remote preflight shows one is
missing, or when the user explicitly asks to reseed the remote base images.

Shared virtualization tests now use base rootfs images plus per-run payload
injection. Individual smoke programs, wrapper scripts, and workload sizes must
not select or create additional rootfs images.

```text
rootfs/rootfs.ext2
  Lightweight comm/query base image. IOCTL semantic smokes inject
  /panthor_ioctl_smoke and /panthor_ioctl_smoke_init into this image before VM
  launch, then pass panthor_ioctl_smoke_mode=<mode> through boot args.

rootfs/rootfs-panfrost.ext4
  GLES/Panfrost userspace base image. GLES smokes inject /root/gles-compute-smoke,
  /root/gpu-smoke.sh, /root/gpu-smoke.env, and /init into this image before VM
  launch, then pass workload arguments through gpu_smoke_args_tokens=...
```

Before injection, the runners prune `firecracker-bins/rootfs/` back to the base
layout shown above. Runtime choices such as IOCTL mode, GLES `--count`,
`--iterations`, and `--warmup` belong in per-run Firecracker config, not in the
rootfs layout.

## Scripts

```text
firecracker-bins/scripts/shared/vmshm-1client/
firecracker-bins/scripts/shared/vmshm-2client/
firecracker-bins/scripts/passthrough/
firecracker-bins/scripts/tools/
```

Launch scripts compute `firecracker-bins/` from their own path and should use
relative semantic paths below that root. Avoid new top-level `.sh` files under
`firecracker-bins/`.

## Test Sources

```text
GPU-SFTP/tests/gpu-compute-smoke/
GPU-SFTP/tests/panthor-ioctl-smoke/
GPU-SFTP/tests/vmshm-test/
```

Source trees belong under `tests/`; built test binaries belong under
`firecracker-bins/bin/` or inside the selected rootfs image.

## DTB Artifacts

```text
GPU-SFTP/artifacts/dtb/
```

Firecracker accepts optional `machine-config.dump_fdt_path`. Set that field to
write a generated DTB into `artifacts/dtb/`; omit it when no dump is needed.
`GPU-SFTP/artifacts/dtb/trans_dtb.sh` converts the canonical DTB in this folder
to DTS.

## Remote Migration

The repo entry scripts call `scripts/lib/gpu_sftp_layout.sh` after rsync. That
helper migrates old remote paths into this layout, including old top-level
binaries, `config/`, `config-2client/`, `kernel-client/`, `kernel-proxy/`,
top-level rootfs images, passthrough `Image.bak*` backups, Panfrost rootfs
build work directories, old test source directories, top-level DTB/DTS files,
the old top-level `trans_dtb.sh`, trace run markers, and the obsolete
`firecracker-image-bins/` directory.
